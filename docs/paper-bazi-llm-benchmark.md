## 二、研究动机与问题定义

### 2.1 核心问题

虚拟角色生成领域存在两大瓶颈：

1. **Prompt 方法的上限**：手工编写的 persona prompt 无法在有限 token 长度内捕捉人类性格与经历的复杂性 (Liu et al., 2023)
2. **微调方法的扩展性**：针对特定角色的 fine-tuning 难以跨多样化人设进行规模化扩展 (Hu et al., 2022; Dettmers et al., 2023)

传统方法（对话树、有限状态机、行为树）成本高、场景脆弱，容易产生模板化人设 (Millington and Funge, 2019)。

### 2.2 论文的核心主张

将八字（BaZi / Four Pillars of Destiny）重新定义为**条件特征生成模型（Conditional Feature-Generation Model）**：

- **不是**玄学预测工具
- **而是**一种文化基础的符号表征系统，将时间离散化为符号属性，关联人格特质和时间动态
- 类比 MBTI 或星座作为"文化词汇表"（Cultural Vocabulary），八字在此作为身份和人生轨迹的叙事表征系统

### 2.3 经验性支撑

论文引用了出生时间与人生结果相关的实证研究，建立"时间基线"（Temporal Baseline）：

| 领域 | 研究 | 发现 |
|------|------|------|
| 教育 | Crawford et al. (2014); UK Biobank Study (2015) | 出生季节/入学相对年龄效应影响学业成绩 |
| 健康 | Boland et al. (2015); Li et al. (2016) | 出生月份与呼吸系统、心血管疾病风险存在系统性关联 |
| 生理 | Hemati et al. (2021); Disanto et al. (2012) | 产前时序与后续生理发育存在生物学机制 |
| 反面参照 | Carlson (1985); Wyman & Vyse (2008); Dean (2025) | 西方占星术指标缺乏预测效度 |

论文的实验逻辑：先用经验验证的出生时间变量建立 LLM 时间基线，再评估八字符号特征能否提供增量信号。

## 三、数据集：Celebrity 50

### 3.1 设计理念

评估角色模拟通常需要完整的人生叙事（difficult to verify）。论文采用 QA 形式，将评估降维为对关键人生事件节点的离散推理，对齐现有推理和常识评估基准 (Talmor et al., 2019)。

### 3.2 数据统计

| 指标 | 数值 |
|------|------|
| 人物数量 | 50 |
| 来源国家 | 29 个（均为北半球） |
| 出生年代 | 约 1940 年前后 |
| 总 QA 对数 | 488 |
| 平均每人题数 | ≈ 9.76 |
| 性别分布 | 37 男 / 13 女 |
| 题目类型 | 多选题（4 选 1） |
| 覆盖维度 | 财富、健康、亲属、事业、情感 |

### 3.3 构建流程

```
精确出生时间（astro.com）
  → Qwen API 检索人物传记（五个维度）
  → Qwen 生成多选题
  → 脚本整合为 JSON 格式
  → 自动化清洗 + 人工审核
```

### 3.4 质量控制（三条剔除规则）

1. 含真实人名/组织名的题目 → 剔除
2. 要求过于具体数字（如精确财富数额）的题目 → 剔除
3. 超出八字合理预测能力的题目 → 剔除

不合格题目由 LLM 迭代优化后替换，最终全部经人工验证。

### 3.5 任务定义

- **输入**：出生日期/时间 + 性别 + 出生地 + 多选题
- **输出**：正确答案选项
- 要求模型结合八字规则和外部世界知识进行推理

## 四、系统架构：BaZi-LLM

### 4.1 总体流程

```
输入（生日、性别、出生地）
  → [Stage 1] BaZi 规则映射（确定性）
  → [Stage 2] 经典逻辑解释（符号推理）
  → [Stage 3] 场景导向分析（领域映射）
  → [Stage 4] 动态人设 Prompt 生成
```

### 4.2 Stage 1：BaZi 规则映射

基于八字理论的规则引擎，将出生信息转换为结构化八字命盘：

- **输入**：年月日时 + 性别 + 出生地
- **输出**：八个天干地支符号 + 关联属性
- 关联属性包含两类：
  - **人格特征**：由五行（木火土金水）平衡及其对应角色推导
  - **日常动态状态**：与健康、事业、财富、亲属关联的时序特征

关键：此阶段为**确定性计算**，不涉及 LLM 推理。

### 4.3 Stage 2：经典逻辑解释

在原始符号特征上进行粗粒度解释推理，包含三大核心组件：

| 组件 | 中文 | 作用 |
|------|------|------|
| Ten Gods | 十神 | 日主与其他干支间的符号关系角色（如正官、偏印、食神等） |
| Shen Sha | 神煞 | 与特定人生倾向或外部影响相关的辅助标记（如天乙贵人、桃花等） |
| Pattern Structures | 格局 | 反映更广泛人格取向的高层符号分组（如伤官格、从格等） |

产出：**条件性解释特征（Conditional Interpretive Features）**，而非确定性结论。

### 4.4 Stage 3：场景导向分析

将八字特征与五大领域模块耦合：

- 健康（Health）
- 事业（Career）
- 财富（Wealth）
- 情感（Relationship）
- 亲属（Kinship）

同一符号特征在不同场景下表现不同。例如"事业心"特征在人际冲突场景 vs. 财务机遇场景中的不同映射。

### 4.5 Stage 4：动态 Prompt 生成

将解释特征整合为动态 prompt，同时编码：

- **长期稳定特质**（Stable Personality Traits）
- **短期时序变化**（Temporal Variations via Flowing Years/Months/Days）

输出时间序列化、环境感知的角色画像。

### 4.6 三大方法论创新

1. **最小输入，丰富输出（Minimal Input, Rich Output）**：仅需出生信息即可产出丰富人设
2. **符号-逻辑整合（Symbolic-Logical Integration）**：规则映射 + 十神/神煞/格局解释逻辑，产出可解释的结构化特征
3. **场景自适应（Scenario Adaptivity）**：人设表征随健康/事业/财富/情感/亲属场景动态变化

### 4.7 技术实现

- 八字知识生成：**DeepSeek-R1** (Guo et al., 2025)
- 推理引擎：**Doubao-1.5-Thinking-Pro**（字节跳动，Mixture-of-Experts 推理模型）

## 五、实验设计与结果

### 5.1 三组对照实验

| 设置 | 说明 |
|------|------|
| Vanilla LLM + BaZi（Baseline） | 直接给 LLM 提供八字特征，不添加推理模块 |
| Vanilla LLM + BaZi Rule Knowledge | 在八字特征基础上增加显式符号知识规则 |
| Our Model（完整系统） | 多智能体架构，完整的符号推理 + LLM 推理 |

另设 **Shuffled Birthday Control（打乱生日控制组）**：将每个人的真实生日替换为另一人的生日，验证系统是否真正依赖出生时间-八字对应关系。

### 5.2 核心实验结果

#### Table 1：Celebrity 50 主实验

| 设置 | DeepSeek-v3 | Gemini-2.5-flash | GPT-5-mini |
|------|-------------|-------------------|------------|
| Baseline | 39.3% | 42.2% | 34.0% |
| +BaZi Rule Knowledge | 35.9% (↓8.7%) | 42.4% (↓4.1%) | 36.9% (↑8.5%) |
| **Our Model** | **51.2% (↑30.3%)** | **47.1% (↑6.6%)** | **55.3% (↑62.6%)** |

**发现**：
- 单纯加八字规则知识效果不稳定（DeepSeek 反而下降 8.7%）
- 完整系统一致优于基线，相对提升 6.6%–62.6%
- GPT-5-mini 基线最弱，但在完整系统中表现最强

#### Table 2：打乱生日实验（Our Model）

| 条件 | DeepSeek-v3 | Gemini-2.5-flash | GPT-5-mini |
|------|-------------|-------------------|------------|
| 真实生日 | 51.2% | 47.1% | 55.3% |
| 打乱生日 | 40.6% (↓20.7%) | 35.5% (↓24.6%) | 30.0% (↓45.7%) |

**发现**：打乱生日后所有模型准确率大幅下降，最高达 45.7%，证明系统确实在利用真实的出生时间-八字符号对应关系。

#### Table 3：Vanilla LLM 打乱对照

| 条件 | DeepSeek-v3 | Gemini-2.5-flash | GPT-5-mini |
|------|-------------|-------------------|------------|
| 真实生日 | 39.3% | 42.2% | 34.0% |
| 打乱生日 | 42.5% (↑8.1%) | 42.1% (↓0.2%) | 34.8% (↑2.4%) |

**发现**：Vanilla LLM 对打乱几乎不敏感，说明它们并未内化八字知识——只是"看了个热闹"，推理不依赖符号对应关系。

### 5.3 竞赛评测

在第 15 届全球算命师锦标赛 2024（香港少年风水师协会主办）真题上，模型达到 **60% 准确率**，相当于当年第三名。

### 5.4 核心结论

1. 完整的符号推理 + LLM 推理架构显著优于纯 LLM 方法
2. 打乱实验证明系统真正依赖八字符号-传记的时间对齐
3. Vanilla LLM 对八字知识的内化非常有限，不能简单"喂数据"
4. 多模型协作（DeepSeek-R1 生成知识 + Doubao 推理）是有效策略

## 六、案例研究：模型差异对比

论文对 Sergey Brin（sergey_brin_P042）案例进行了三模型对比：

### 6.1 理论解释差异

| 模型 | 格局判定 | 风格 |
|------|----------|------|
| DeepSeek-V3 | 伤官格 | 保守、规则驱动 |
| Gemini-2.5-flash | 伤官格 | 保守、稳定 |
| GPT-5-mini | 从儿格 | 大胆、灵活、探索性 |

格局判定的分歧导致了喜忌用神和大运方向的完全不同结论。论文指出这类决策通常依赖专业命理师的经验判断。

### 6.2 场景映射差异

| 模型 | 推理模式 |
|------|----------|
| DeepSeek-V3 | "特征→预测"，刚性直线推理，易受局部信息偏差影响 |
| Gemini-2.5-flash | 多维度整合，全面整体分析 |
| GPT-5-mini | 最接近人类命理师，动态适应用户当前人生语境，主动探索替代场景 |

### 6.3 输出表达差异

| 模型 | 表达风格 | 风险 |
|------|----------|------|
| DeepSeek-V3 | 绝对陈述（"一定会…"） | 预测命中时显得精准，但失准时严重损害信任 |
| Gemini-2.5-flash | 概率性语言（"可能""较大概率"） | 稳健但可能缺乏冲击力 |
| GPT-5-mini | 概率性表达 + 多种可能并存 | 最接近人类顾问的表达方式 |

### 6.4 共同点

- 给定相同上游结果后，推理路径趋同
- 所有模型具备基本的八字知识，足以完成一般咨询
- **三个模型均缺乏反思和自我纠错机制**

### 6.5 综合评价

| 阶段 | 最佳模型 | 原因 |
|------|----------|------|
| 理论推理 | Gemini-2.5-flash | 最稳定、最抗噪声干扰 |
| 最终输出 | GPT-5-mini | 生成最接近人类命理师风格的解释 |

## 七、论文局限性与风险

### 7.1 数据局限

| 局限 | 详细说明 |
|------|----------|
| LLM 生成偏差 | 叙事和问题由 Qwen API 生成，存在幻觉、偏见和事实错误风险 |
| 数据规模不足 | 仅 50 人、488 题，统计效力有限 |
| 性别不平衡 | 37 男 / 13 女，可能引入性别偏差 |
| 出生信息准确性 | astro.com 数据仍可能有误 |
| 文化/时代偏差 | 主要是 1940 年代出生的西方名人，不一定推广到其他时代或文化 |

### 7.2 方法论风险

| 风险 | 说明 |
|------|------|
| 循环依赖 | LLM 生成问题 + LLM 检索传记 + LLM 回答 → LLM 可能从训练数据中"记住"名人信息 |
| 逻辑跳跃 | 从"出生月份影响健康"（流行病学）到"八字系统有效"之间的推理链条不完整 |
| 科学有效性争议 | 八字的理论基础在现代科学框架下缺乏验证 |

### 7.3 适用性判断

论文的价值不在于证明八字的科学有效性，而在于：
- 建立了首个可量化评测的八字推理基准
- 验证了符号推理 + LLM 的混合架构在文化推理任务上的有效性
- 提供了将传统文化系统与 AI 结合的工程化方法论

## 八、对吉祥物项目的技术启示

> 以下分析基于论文发现，结合 [八字吉祥物 3D 设计方案](plans/2026-02-10-bazi-mascot-3d-design.md) 进行对照。

### 8.1 八字计算模块的启示

论文的 BaZi Rule Mapping 对应项目中的 `calculate_bazi` tool。当前设计产出"八字、五行、喜用神"。论文实验表明，仅有五行信息不够（Vanilla LLM + BaZi 效果有限），需要更深层的符号解读：

| 论文层级 | 含义 | 对吉祥物项目的作用 | 当前设计是否覆盖 |
|----------|------|---------------------|------------------|
| 基础命盘 | 八字四柱、五行比例 | 吉祥物主色调/材质基底 | ✅ 已有 |
| 喜用神 | 命局需要的五行 | 吉祥物的"补运"核心元素 | ✅ 已有 |
| 十神 | 日主与其他干支的关系角色 | 吉祥物的"性格气质"（食神→圆润可爱 vs. 七杀→锐利凌厉） | ❌ 缺失 |
| 神煞 | 辅助标记（天乙贵人、桃花等） | 吉祥物的"装饰元素"（桃花→花卉，文昌→书卷） | ❌ 缺失 |
| 格局 | 高层人格取向 | 吉祥物的"整体风格基调"（从官格→庄重 vs. 食伤格→灵动） | ❌ 缺失 |

**建议**：扩展 `calculate_bazi` 的输出结构，增加十神、神煞、格局三个层级。

### 8.2 Agent 推理架构的启示

论文最关键的实验发现：**符号推理的层次越深、结构越完整，LLM 的下游表现越好**。

三组对照实验直接映射到 Agent 设计策略：

| 论文设置 | 准确率 | 对应 Agent 策略 | 推荐 |
|----------|--------|----------------|------|
| Vanilla LLM + BaZi 特征 | 34-42% | 只把 calculate_bazi 结果交给 DeepSeek，让它自由发挥 | ❌ |
| + BaZi Rule Knowledge | 35-42% | 在 system prompt 里加八字规则解释 | ❌ 效果不稳定 |
| **Full Model（多阶段推理）** | **47-55%** | 分步推理：命盘 → 解读 → 场景映射 → 设计输出 | ✅ |

当前设计中 Agent 的流程是：
```
输入生日 → calculate_bazi → Agent 解读并设计 → generate_model
```

论文建议改为链式推理：
```
输入生日 → calculate_bazi（结构化数据，含十神/神煞/格局）
  → Agent Step 1：命盘解读（符号推理，解释各层级含义）
  → Agent Step 2：视觉映射（将符号特征翻译为设计语言）
  → Agent Step 3：吉祥物描述（输出完整的 3D Prompt）
  → generate_model
```

### 8.3 符号-视觉映射表

论文的 Scenario-Oriented Analysis 将八字特征按领域解读。吉祥物项目的"场景"是视觉设计。需要建立一张**八字特征 → 视觉设计语言**的映射表：

#### 五行 → 色彩与材质

| 五行 | 色彩 | 材质质感 | 形态语言 |
|------|------|----------|----------|
| 木 | 翠绿、青色 | 温润、木纹 | 向上生长、有机弧线 |
| 火 | 红、橙、暖色渐变 | 光泽、半透明 | 尖锐、火焰轮廓、动态 |
| 土 | 黄、棕、米色 | 陶瓷、磨砂 | 厚重、圆润、稳定 |
| 金 | 白、金、银 | 金属光泽、抛光 | 几何、棱角、精致 |
| 水 | 蓝、黑、深色 | 透明、流动感 | 柔软、波浪、流线 |

#### 十神 → 气质与姿态

| 十神 | 性格特征 | 吉祥物气质 | 姿态建议 |
|------|----------|------------|----------|
| 正官 | 端正、守规 | 庄重、对称 | 端坐、直立 |
| 偏官（七杀） | 霸气、果断 | 锐利、力量感 | 战斗姿态、展翅 |
| 正印 | 温和、包容 | 慈爱、环抱 | 怀抱或托举 |
| 偏印 | 神秘、独特 | 异形、奇幻 | 悬浮、冥想 |
| 食神 | 乐观、享受 | 圆润、可爱 | 嬉戏、微笑 |
| 伤官 | 创新、叛逆 | 夸张、灵动 | 跳跃、不对称 |
| 比肩 | 独立、竞争 | 挺拔、刚毅 | 昂首、并肩 |
| 劫财 | 果敢、冲动 | 速度感 | 奔跑、冲锋 |
| 正财 | 勤勉、务实 | 饱满、充实 | 手持宝物 |
| 偏财 | 慷慨、冒险 | 华丽、开放 | 张开双臂 |

#### 神煞 → 装饰元素

### 8.4 打乱生日实验的质量验证启示

论文用"打乱生日"实验证明系统确实依赖真实的符号-传记对应关系。项目可借鉴此方法做开发阶段的质量测试：

**测试方案**：同一用户分别用真实生日和随机生日生成吉祥物 → 检查设计差异是否显著且合理。若随机生日也能产出"看似合理"的结果，说明 Agent 并没有真正利用八字信息。

### 8.5 模型选型的启示

论文对三个 LLM 的对比分析提供直接参考：

| 模型 | 论文评价 | 对项目的建议 |
|------|----------|-------------|
| DeepSeek-V3 | 刚性确定性，"特征→预测"直线模式 | 当前项目选用 DeepSeek，需在 prompt 中显式要求多方案/概率性表达 |
| GPT-5-mini | 最灵活，最像人类顾问 | 备选方案，预算允许时可考虑 |
| Gemini-2.5-flash | 最稳定、抗噪声 | 适合做"质量兜底"或审核环节 |

论文的多模型协作策略（DeepSeek-R1 生成知识 + Doubao 推理）也值得参考：可用一个模型做八字深度解读，另一个做创意设计。

### 8.6 DeepSeek 使用注意事项

论文案例分析表明 DeepSeek-V3 的风格特点：

- 倾向"绝对陈述"——在吉祥物设计中可能导致过于死板的描述
- "特征→预测"的刚性推理——可能跳过创意探索直接给出单一方案
- 缺乏自我反思——不会主动质疑或调整自己的设计方案

**应对**：在 system prompt 中显式要求：
1. 提出 2-3 种可选的吉祥物设计方向
2. 用概率性语言解释八字对设计的影响
3. 在输出设计方案前，先自我审视八字解读的合理性

## 九、行动建议

### 高优先级（直接影响系统效果）

1. **扩展 `calculate_bazi` 输出结构**
   - 当前：八字 + 五行 + 喜用神
   - 建议增加：十神分布 + 神煞列表 + 格局判定
   - 这是论文验证的核心发现：深层符号特征提供关键增量信号

2. **Agent 改为链式推理**
   - 拆分为：命盘解读 → 视觉映射 → 设计描述，三个显式推理步骤
   - 每步在对话中可见，用户可以在任一节点介入纠偏

3. **建立八字-视觉映射表并写入 system prompt**
   - 将上述映射表作为 Agent 的参考知识
   - 确保 Agent 有明确的"翻译规则"，而非自由联想

### 中优先级（提升质量与可信度）

4. **开发阶段实施"打乱生日"测试**
   - 同一批测试用例用真实/随机生日各跑一遍
   - 对比生成结果差异，验证 Agent 是否真正利用八字信息

5. **引导 DeepSeek 多方案输出**
   - 在 system prompt 中要求提出 2-3 种吉祥物方向
   - 避免 DeepSeek 的"绝对陈述"倾向

### 低优先级（长期优化方向）

6. **考虑多模型协作**
   - 参照论文用 DeepSeek-R1 生成知识 + Doubao 推理的策略
   - 可用一个模型做八字深度解读，另一个做创意设计

7. **流年动态融入设计**
   - 论文强调的"时间动态性"可作为增值功能
   - 在底座刻字或设计主题中融入当年运势元素
